Functional Requirement:
-----------------------
- Start the checkout process, lock cart, prepare summary
- Apply a coupon or promotion code
- Select shipping option
- Submit payment details
- Finalize and place the order
- Get the current checkout summary for review
- (Optional): Retrieve past checkouts

Non-functional Requirement:
---------------------------
- Low latency
- High availability
- Massive scale
- (and so on...)

Assumptions:
------------
- Suppose we expect 10k checkouts / sec during peak sale events
- Each checkout might generate 4 - 5 API requests. So about 40K - 50K RPS
- Inventory and payment are bottlenechs and need to be heavily parallelized
- During off-peak hours: load may drop to 1K RPS
- We need auto-scalling and rate limiting to manage surges
