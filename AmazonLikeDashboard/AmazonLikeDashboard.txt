Functional Requirement:
-----------------------
- Fetch the dashboard data for the current user/session
- Add a new widget to the user's dashboard
- Remove a widget from the dashboard
- Fetch alert notifications
- Save updated dashboard layout
- Fetch metric data (orders, revenue, inventory, etc.)
- View access / change logs for auditing


Non-functional Requirement:
---------------------------
- Low latency
- High availability
- Massive scale
- (and so on..)


Assumptions:
------------
- 23 M users, 500k concurrent sessions at peak
- 50k req per sec for dashboard refresh
- write req: 20 req / sec
- Spike reads: 200k req / sec
- Use auto-scaling horizontally


API Design:
-----------
GET /dashboard
POST /dashboard/widget
DELETE /dashboard/widget/{widgetId}
GET /dashboard/alerts
POST /dashboard/layout
GET /metric/{type}
GET /auditing


Data Model / DB Schema:
-----------------------
- Primary data store: NoSQL (CassandraDB, or others): (userId, layout, widgets, listUpdated)
- Metric data store: Time-series (influxDB, etc.): (eventId, metricType, value, timestamp)
- RDBMS (optional): (userId, role, status)
- Cache (Radis, etc.)


HLD:
-----
Client -> API Gateway -> Microservices -> DB + Cache
