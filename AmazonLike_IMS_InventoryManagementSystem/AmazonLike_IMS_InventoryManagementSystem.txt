Functional Requirement:
-----------------------
- Add product
- Get product details and inventory status
- Update product information
- Remove the product
- Get inventory for all products in a warehouse
- Rserve inventory for an order
- Release reservation (on order cancellation..)
- Add stock after shipment or return
- Get stock levels across all locations for a product
- Get audit trail fof inventory changes


Non-Functional Requirement:
---------------------------
- Low latency
- High availability
- Massive scale
- (and so on...)


Assumptions:
------------
- Traffic Estimation:
-- 10M products and 1000 warehouses with 10k concurrent users and each triggers 10 requests per minute during peak
--- Peak QPS = 10k user * 10 req / min = 1666 QPS
--- Reads > writes: easily surpassing 10k QPS
--- Bulk updates spikes separately: batch processing

- Load Handling:
-- Load balancers, horizontal scaling for API servers
-- Separate read replicas
-- Queue writes for handling batch updates or audits
